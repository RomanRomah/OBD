# Electronics Store Database 

Цей репозиторій містить навчальний проєкт з проєктування та реалізації реляційної бази даних у межах курсу **“Вступ до баз даних”**.

Предметна область проєкту — автоматизація роботи інтернет-магазину електроніки (керування товарами, замовленнями, клієнтами та відгуками).

---

## Мета проєкту
- Аналіз предметної області роздрібної торгівлі гаджетами.
- Побудова логічної та фізичної моделі БД на базі PostgreSQL.
- Реалізація сучасної взаємодії з даними через Prisma ORM та Node.js.

---

## Опис предметної області
Система призначена для ведення обліку в магазині електроніки:
- **Товари та бренди**: зберігання характеристик гаджетів та їх виробників.
- **Клієнти та логістика**: база покупців із прив'язкою до адрес доставки.
- **Замовлення**: фіксація покупок та їхнього вмісту.
- **Зворотній зв'язок**: система рейтингів та текстових відгуків клієнтів.

---

## Використані технології
- **PostgreSQL** — основна реляційна СУБД.
- **Prisma ORM** — інструмент для керування схемою та міграцій.
- **Node.js** — середовище для виконання скриптів взаємодії з БД.
- **Docker** — для розгортання контейнера з базою.

---

## Лабораторні роботи

### **ЛР-1 — Аналіз предметної області та ER-діаграма**
- Визначено основні сутності: `products`, `brands`, `customers`, `orders`.
- Побудовано ER-діаграму зв’язків між товарами та категоріями.

### **ЛР-2 — Проєктування реляційної схеми**
- Описано типи даних для полів (VARCHAR, INT, TIMESTAMP).
- Встановлено обмеження цілісності (Primary Keys, Foreign Keys, Unique).

### **ЛР-3 — Маніпулювання даними (OLTP)**
- Написано SQL-скрипти для заповнення бази початковими товарами та клієнтами.
- Реалізовано базові запити на додавання та оновлення статусів замовлень.

### **ЛР-4 — Складні запити та аналітика (OLAP)**
- Написано запити з використанням `JOIN`, `GROUP BY` та агрегатних функцій.
- Розраховано загальний прибуток магазину та статистику продажів по брендах.

### **ЛР-5 — Нормалізація бази даних**
- Проведено перевірку схеми на відповідність 1НФ, 2НФ та 3НФ.
- Винесено адреси клієнтів в окрему таблицю для уникнення надмірності.

### **ЛР-6 — Міграції та Prisma ORM**
- Налаштовано **Prisma Client** для зв'язку Node.js із PostgreSQL.
- Виконано міграцію для створення таблиці `review` (відгуки).
- Проведено модифікацію існуючих таблиць (додано поштовий індекс, телефон та видалено країну бренду).
- Написано скрипт `test.js` для автоматизованої перевірки вставки даних.

---

## Припущення та обмеження
- Кожен товар належить лише до одного бренду.
- Видалення клієнта призводить до каскадного видалення його адрес та відгуків.
- Проєкт має навчальний характер і не є промисловим рішенням.
